# CE-planner 外部候选规划器运行模式改造

## 修改目标

新增两种可切换运行模式，保持候选问题生成、计划解析、动作映射和 CounterCNF/Kissat 验证语义不变。

- `native`（默认）：求解器在各自原始程序目录执行；本轮 refined PDDL、控制台输出和规范化计划固定写入 `runs/current`，下一轮覆盖。
- `isolated`：完整保留原有 `runs/run_<pid>/iter_xxxx/planner_work` 调试流程。

参数：

```bash
IGC_CANDIDATE_RUNTIME_MODE=native
IGC_CANDIDATE_RUNTIME_MODE=isolated
```

兼容别名：`original`、`solver_dir`、`solver-dir` 等价于 native；`workspace`、`debug` 等价于 isolated。

## 修改文件

1. `lama/search/cegis_native_runtime.inc`
   - 完整新增 T1、CFF、CNF/DNF/PIP、CPAH、iGC、GC-LAMA、gCPCES 的原目录执行实现。
   - iCPCES 复用现有原目录执行实现。
2. `lama/search/cegis_framework.cc`
   - 由严格补丁脚本加入运行模式枚举、参数解析、runner 分派和固定目录选择。
   - 现有 isolated 方法不删除、不改写。
3. `lama/search/Makefile`
   - 为 `cegis_native_runtime.inc` 增加显式依赖，保证以后只修改 inc 文件也会重新编译 `cegis_framework`。

## native 模式数据流

```text
反例集
  -> runs/current/refined_<solver>_problem.pddl
  -> 必要时 runs/current/refined_<solver>_domain.pddl
  -> cd 外部求解器原目录并从 PDDL 重新执行
  -> 求解器最终计划文件或本轮 planner.stdout
  -> runs/current/planner.result 或 plan-result
  -> ActionMapper
  -> runs/current/candidate.plan
  -> CounterCNF/Kissat 完整原问题验证
```

求解器自己的 translator、preprocess、SICStus、c2d 等中间产物留在原目录，由下一轮覆盖。调用前只主动删除可能被误读的旧最终计划文件。

## 安装

在 CE-planner 仓库根目录解压本包，然后执行：

```bash
python3 tools/apply_candidate_runtime_mode.py --repo .
./build
```

补丁脚本先检查全部源码锚点，再同时写入；版本不匹配时拒绝修改。首次执行会生成：

```text
lama/search/cegis_framework.cc.bak-native-runtime
lama/search/Makefile.bak-native-runtime
```

## 使用

默认 native：

```bash
./plan-cegis-t1 domain.pddl problem.pddl result.plan false
```

显式 native：

```bash
IGC_CANDIDATE_RUNTIME_MODE=native \
./plan-cegis-dnf domain.pddl problem.pddl result.plan false
```

恢复原有全量调试目录：

```bash
IGC_CANDIDATE_RUNTIME_MODE=isolated \
./plan-cegis-t1 domain.pddl problem.pddl result.plan false
```

native 模式默认目录为 `runs/current`。设置 `IGC_CEGIS_RUN_DIR` 时，该值是实际固定工作目录，不会再自动追加 `/current`。

## 回退

```bash
tools/rollback_candidate_runtime_mode.sh .
./build
```

## 已执行测试

1. 新增 903 行 native C++ 实现使用 `g++ -std=gnu++98 -Wall -Wextra -pedantic -fsyntax-only` 通过。
2. 严格补丁器：首次修改成功；第二次执行保持幂等；源码和 Makefile 备份生成正确。
3. shell 集成模拟覆盖：
   - T1
   - CFF
   - DNF
   - CPAH
   - iGC
   - GC-LAMA
   - gCPCES
   - iCPCES 分派
4. 所有模拟规划器记录的当前目录均为各自原程序目录。
5. `runs` 下只有固定 `current`，没有 `iter_xxxx` 或 `planner_work`。
6. 规划器结果成功写回 `runs/current`。
7. T1 旧结果防误读测试通过：本轮进程成功但不产生新计划时，原目录旧 `planner.result` 和 runs 中旧结果均已先删除，返回解析错误而非复用旧计划。

可重新运行随包测试：

```bash
tests/run_simulated_native_runtime_tests.sh
```

## 尚需在目标机器执行的真实回归

容器中没有完整 CE-planner 依赖和各真实外部求解器运行环境，因此无法代替目标 Ubuntu 机器完成真实 benchmark 回归。应用后至少执行：

```bash
./build
IGC_CEGIS_SELFTEST=1 ./lama/search/release-search
```

然后分别用 native 和 isolated 对同一小测例测试，确认最终计划和反例迭代一致。建议先测试 blocks/b2 或当前已知能快速通过的 dispose 小测例。
